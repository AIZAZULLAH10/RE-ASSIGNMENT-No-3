Tool 1: Selenium  
Selenium is primarily used for automated testing  of web applications, but it can also play a 
valuable role in reverse engineering legacy web systems —especially when documentation is 
missing , source code is unavailable , or business logic is only accessible via the UI . Selenium 
helps you automate actions (clicks, inputs) in the browser, so you can observe how the system 
behaves and extract the hidden logic  from the user interface.

Step 2: Open Selenium and Start Automating  
Here’s a basic Python script to launch the browser and open the system:  
from selenium import webdriver  
from selenium.webdriver.common.by import By  
import time  
 
# Start browser

# Let  it load

Tool 2: Docker  
Docker is a containerization tool  that helps package applications and their dependencies into 
isolated containers. When you're working with a legacy system , Docker can be used in reverse 
engineering  to help replicate, analyze, and modernize  the system —especially when the 
original environment is outdated or undocumented. Use Docker to capture, reproduce, and 
explore  how a legacy system works —without breaking the original setup.  
Docker is a containerization tool  that helps package applications and their dependencies into 
isolated containers. When you're working with a legacy system , Docker can be used in reverse 
engineering  to help replicate, analyze, and modernize  the system —especially when the

Prerequisites  
Step 0: – What You Need  
Tool  Description  
Docker  Install Docker Desktop from https://www.docker.com/products/docke r-desktop  
Access  You need access to the legacy app’s files or server (even partially: code, configs, 
binaries, logs)  
Basic 
Knowledge  Terminal/Command Line, Linux basics (if possible)  
Step -by-Step  Process  
Step 1: Understand the Legacy Environment  
Before doing anything:  
 Ask: Where is the app hosted (Linux, Windows, on -premise)?  
 What components are involved?  
o Web server (Apache, Nginx)?  
o Database (MySQL, Oracle)?  
o Any scripts or compiled code (PHP, Java, .exe)?  
Example:  
Legacy system includes:  
 PHP app  
 Apache server  
 MySQL 5.5 database  
 Running on Ubuntu 14.04  
 
Step 2: Copy the Environment  

Try to get these from the legacy system:  
 Application files (e.g., PHP, HTML, JS, etc.)  
 Config files (httpd.conf, .env, my.cnf, etc.)  
 Database dump (e.g., .sql file)  
 
Step 3: Cr eate a Dockerfile (to Rebuild Environment)  
A Dockerfile defines the OS and environment needed.  
Example Dockerfile for a PHP -Apache legacy app:  
FROM php:5.6 -apache   # match legacy PHP version  
 
COPY . /var/www/html/ # copy app files  
 
RUN docker -php-ext-install mysqli  # install MySQL driver  
 
Step 4: Create a docker -compose.yml (for full stack)  
If the app uses a database or multiple services, use Docker Compose  to define everything.  
Example docker -compose.yml:  
version: '3'  
services:  
  web:  
    build: .  
    ports:  
      - "8080:80"  
    volumes:  
      - .:/var/www/html  
  db: 
    image: mysql:5.5  
    environment:  
      MYSQL_ROOT_PASSWORD: root  
      MYSQL_DATABASE: legacydb  
    ports:  
      - "3306:3306"  
 
Step 5: Add a Database Dump (Optional)  
Put your .sql  file (database dump) in the db service using a Docker volume or command:  
    volumes:  
      - ./init.sql:/docker -entrypoint -initdb.d/init.sql  
 
Step 6: Run the Legacy System in Docker  
In the terminal, go to your project folder and run:  
docker -compose up --build  
This will:  
 Build the PHP -Apache environment  
 Start MySQL  
 Load your legacy app  
 Expose it at: http://localhost:8080  

Step 7: Explore the Running System  
Once it’s up, now you can:  
Action  Use it to…  
View app in browser  Test how the legacy system behaves  
Modify files  Test impact of code/config changes  
Log outputs  Check logs for hidden logic or errors  
Run in isolation  No risk of breaking the real server  
 
Step 8: Reverse Engineer Behavior  
While inside the container, you can now:  
 View installed packages ( dpkg -l, php -v) 
 Explore file structures  
 Trace business logic (by following the code)  
 Use tools like tcpdump or curl to analyze internal APIs  
 
Step 9: Document Findings  
As you explore:  
 Write down environment details (OS, dependencies)  
 Note any hardcoded pa ths or credentials  
 Record app behavior based on input/output  
You can later use this info to migrate or refactor  the system.

Tool 3: Apache Nifi  
Apache NiFi  is a powerful data flow automation tool used to collect, route, transform, and 
deliver data. When you're reverse engineering a legacy system , NiFi can help visually trace 
how data flows  in and out of the system —especially when dealing with legacy databases, APIs, 
files, or message queues.  Use Apache NiFi to reverse engineer data flow in a legacy system by 
visually mapping, monitoring, and analyzing  how data moves across systems.  
Prerequi sites: Step 0  
Item  Description  
Java Installed  NiFi runs on Java (JDK 8+ recommended)  
Apache NiFi  Download from: https://nifi.apache.org/download.html  

Item  Description  
Basic Concepts  Understand source/destination systems (e.g., DB, FTP, APIs)  
Access  You need access to where the data lives: file system, DB, API, etc.  
Step -by-Step  process : 
Step 1: Download and Start NiFi  
1. Download NiFi (binary release):  
https://nifi.apache.org/download.html  
2. Unzip it and start:  
3. ./bin/nifi.sh start  
4. Open in browser:  
5. http://localhost:8080/nifi  
Step 2: Understand Legacy Data Sources  
Identify what the legacy system uses:  
 Files (CSV, XML, JSON)  
 Relational DB (MySQL, Oracle, etc.) 
 REST APIs  
 Message queues (Kafka, JMS)  
Example:  
 Legacy app stores data in old_data.csv  
 Sends updates via HTTP to a client system  
 Uses MySQL 5.5 as backend  
Step 3: Use NiFi Processors to Tap Into the Data  
Create a data flow pipeline  using NiFi’s  drag-and-drop UI.  
Data Source  NiFi Processor  
Files  GetFile , ListFile  
DB ExecuteSQL , QueryDatabaseTable  
HTTP API  InvokeHTTP , ListenHTTP  
FTP/SFTP  GetSFTP , GetFTP  
 
Step 4: Trace Data Movement and Structure  
As you pull data from legacy systems:  
 Use LogAttribute  to print what data looks like.  
 Use PutFile  or PutDatabaseRecord  to capture it.  
 Use RouteOnAttribute  or EvaluateJsonPath  to break down fields.  
Example Flow:  
GetFile (CSV) ➜ SplitText ➜ ConvertRecord ➜ LogAttribute ➜ PutDatabaseRecord  
This lets you: 
 Read the file  
 Break it into rows  
 Convert it to structured form  
 Log and observe the structure  
 Push it into a modern database (optional)  

Step 5: Analyze and Document the Logic  
NiFi allows you to inspect : 
 File structure  
 Database schema  
 API behavior (input/output)  
 Triggers or schedules  
Document:  
 What kind of data is being processed?  
 Where is it coming from?  
 How is it transformed or filtered?  
 Where is it going?  
This becomes your functional and data flow documentation  of the legacy system.  
Step 6: Add D ebugging or Testing Flows  
NiFi has useful tools for debugging:  
 Use LogAttribute  to see flowfile content  
 Use ReplaceText  or UpdateAttribute  to simulate transformations  
 Use PutFile  to export all intermediate outputs  
This lets you test:  
 How data is structured  
 Whether flows break under malformed input  
 Hidden behaviors in old systems (like triggers)  
Step 7: Export and Share Your Flow  
You can export your NiFi flow as a template:  
 Right -click the canvas → Create Template  
 Save the .xml file 
 Share with others to repl icate the reverse -engineered setup  
Bonus: NiFi Registry for Versioning  
Install Apache NiFi Registry  to: 
 Version control your flows  
 Compare changes  
 Document history of discovery

relevant users (students, parents, and the course coordinator) should automatically receive 
notifications.
